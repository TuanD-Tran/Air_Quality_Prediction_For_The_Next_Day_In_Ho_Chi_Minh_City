# **Hướng Dẫn**

---

## **Mục đích của Drive và Github**
- **Drive:** Chứa các folders outputs, figures,... (Cái này để mình tổng hợp nha - không cần quan tâm) và dữ liệu chính, dữ liệu Train - Test.
- **Github:** Chứa notebooks.

---

## **Workflow**
Mình sẽ làm việc theo từng Branch riêng, cá nhân nha. Không Pull Request lên Branch chính:
1. **Mở file từ GitHub:** Trong Colab, chọn File -> Open notebook, chọn tab GitHub. Nhập URL của repo vào để tải file notebook gốc về.
2. **Tải dữ liệu:** Vẫn làm như cũ là dữ liệu gốc mình sẽ giữ và share cho mọi người.
3. Mỗi người tự code và thử nghiệm trên Colab của mình.
4. **Lưu code về GitHub (Save & Commit):** Khi đã code xong một phần nào đó, chọn File -> Save a copy in GitHub.
**Quan trọng:** Ở hộp thoại hiện ra, hãy chọn tạo một nhánh mới (New Branch), **đừng lưu thẳng đè lên nhánh main.**
5. **Tạo Pull Request (PR):** Lên giao diện GitHub, tạo một PR để gộp nhánh cá nhân vào nhánh main.
Mình sẽ tổng hợp lại và khi ổn mình sẽ Merge vào Main Branch, vì khi làm riêng nó sẽ dễ bị sung đột nên để mình tổng hợp lại và Merge một lần một.
6. **Lưu ý:** Nhớ Clear Output trước khi Save để cho nó gọn nha.

---

## **Đơn giản hơn**
Mọi người có thể tải notebooks về để làm cá nhân sau đó Push lên Github (Tạo folder riêng của mình) 
và mình sẽ tải về và tổng hợp lại.
